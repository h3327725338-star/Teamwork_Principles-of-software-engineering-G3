Project Description:

The University Clinic Appointment System is a web-based application designed to improve healthcare access on campus. It supports SDG 3 – Good Health and Well-Being by providing a structured and efficient appointment booking process.
The system allows students to register, log in, view available time slots, book appointments, reschedule, or cancel appointments. Students can also receive confirmation notifications after booking. Clinic staff can manage appointment schedules and monitor daily bookings.
The main problem addressed is long waiting time and inefficient manual booking processes. Currently, students may need to queue physically or rely on limited communication channels. This system reduces waiting time, improves schedule transparency, and ensures better time management for both students and clinic staff.
The value of this system lies in improving service efficiency, reducing overcrowding at the clinic, and promoting proactive health management among university students.

Overview of SDLC Work:

We followed an Agile methodology using Scrum practices across four weeks.
During the Planning phase, we defined the product vision aligned with SDG 3, identified stakeholders, and created a structured Product Backlog in Trello. User stories were written clearly with priorities and acceptance criteria.
During the Analysis phase, we developed a Use Case Diagram following proper UML notation to represent functional interactions between students and clinic staff. Requirements were refined based on feasibility and scope.
During the Design phase, we created wireframes and interface sketches focusing on user-friendly navigation and simple workflow. Design artifacts were stored and updated in the GitHub design folder with meaningful commit messages.
During Prototype Development, we built an interactive prototype to demonstrate core features such as appointment booking and schedule management. Development progress was tracked through regular GitHub commits linked to Trello tasks.
During Testing and Evaluation, we defined clear test goals and tasks. We conducted usability testing with selected users, gathered feedback, identified issues, and improved the prototype accordingly.
Throughout the project, Trello was used to manage sprints, track task movement (To Do → Doing → Done), and monitor progress through sprint reviews and retrospectives. GitHub commit history shows continuous development and collaboration.
